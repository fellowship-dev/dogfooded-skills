# popsicle

Agent-native onboarding doc generator. Validates docs with fresh agent sessions, builds coverage maps, and emits adapter files for Cursor, Copilot, and Codex.

## What it does

Popsicle scans your repo, finds what matters (entry points, APIs, env vars, architecture), then tests whether your docs actually communicate that to a cold agent. It spawns fresh `claude -p` sessions that can only read your docs — if they can answer questions about the codebase, the docs pass. If not, popsicle fills the gaps and retests.

It also produces persistent artifacts: a coverage map showing documented vs. undocumented areas, a health baseline organized by category, auto-extracted docs for env vars and API routes, and agent adapter files so Cursor, Copilot, and Codex can onboard the same way Claude does.

## Why

Docs rot silently. A fresh agent dropped into a repo with stale or incomplete docs has to re-discover everything from scratch — and may make wrong assumptions it can't verify. Popsicle creates a feedback loop: the docs are the test, a fresh agent session is the student, and the score tells you exactly what's broken.

No rubric. No config. Popsicle figures out what should be documented by reading the code.

## Install

```bash
npx skills add fellowship-dev/dogfooded-skills/skills/ops/popsicle
```

## Usage

**One-shot** — single pass, produces a report:
```bash
/popsicle
```

**Loop** — iterate until pass rate >= 80% (or 5 iterations max):
```bash
/popsicle --loop
```

**With agent adapters** — also generate Copilot and Cursor adapter files:
```bash
/popsicle --agents
```

**Canonical AGENTS.md** — use AGENTS.md as the source of truth instead of CLAUDE.md:
```bash
/popsicle --canonical agents
```

Auto-detection: if `.cursor/` or `.github/copilot-instructions.md` already exists in the repo, `--agents` behavior activates automatically.

## What gets generated

| File | Description |
|---|---|
| `docs/coverage-map.md` | Hierarchical map of documented vs. undocumented areas |
| `docs/health-baseline.md` | Doc health by category (not scores) |
| `docs/generated/env-vars.md` | Auto-extracted environment variables |
| `docs/generated/api-routes.md` | Auto-extracted API routes (if applicable) |
| `docs/generated/db-schema.md` | Schema summary (if ORM/migrations exist) |
| `docs/popsicle-report-YYYY-MM-DD.md` | Per-run results table and gap list |
| `AGENTS.md` | Symlink to `CLAUDE.md` for Codex compatibility (always) |
| `.github/copilot-instructions.md` | Pointer to CLAUDE.md (with `--agents`) |
| `.cursor/rules/project.mdc` | Pointer to CLAUDE.md in Cursor format (with `--agents`) |

Generated files include a `<!-- generated by popsicle -->` header. Popsicle never overwrites files with custom content.
